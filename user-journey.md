Creating a user in Microsoft 365 begins with an administrator signing into the Microsoft 365 Admin Centre at admin.microsoft.com.

The administrator navigates to the **Users section** and selects **Active Users**, then clicks **Add a User**. 

The administrator fills in the required details, including the _user's first name_, _last name_, _display name_, and _username_, which forms the user's email address within the organisation's domain. 

A **password** is set for the new user, either _manually_ or _auto-generated_, and the user can be prompted to change it upon first sign-in. 

Once the basic details are confirmed, the administrator proceeds to the **licence assignment step** during the same setup process.

To assign a licence, the administrator selects the appropriate Microsoft 365 subscription plan available to the organisation, such as **Microsoft 365 Business Basic, Standard, or Premium.**
The specific apps and services included in that licence are then made available to the user upon account activation. 

The administrator completes the setup by **reviewing the user's details** and clicking **Finish Adding**, after which the user receives a welcome email with their login credentials. 

Assigning a licence in Microsoft 365 ensures that the user gains access to the full suite of tools and services included in the chosen plan, such as **Teams**, **OneDrive**, **Word**, **Excel**, and **Outlook**. 

A licence also enables the user to install Microsoft 365 applications on their devices, depending on the plan. From an organisational perspective, licence assignment allows administrators to control access, manage costs, and ensure that each user only has access to the tools relevant to their role. It also ensures compliance with Microsoft's usage policies, as unlicensed users cannot access paid services.
